MongoDB for Absolute Beginners

Introduction

A database Management System provides the mechanism to store and retrieve data. There are different kinds of database Management Systems:

  • RDBMS (Relational Database Management Systems)
  • OLAP (Online Analytical Processing)
  • NoSQL (Not only SQL)

Introduction to NoSQL

Limitations of RDBMS

1. In relational databases, we need to define the structure and schema of data first and then only we can process the data.

2. Relational database systems provide consistency and integrity of data by enforcing ACID properties. However, these properties can cause significant performance overhead and make your database response very slow.

3. Most applications store their data in JSON format, and RDBMS doesn't provide a better way of performing operations such as create, insert, update, delete on this data.

What is NoSQL Database?

NoSQL or non-relational databases provide a mechanism for storage and retrieval of data other than the tabular relations model used in relational databases. NoSQL database doesn't use tables for storing data; it is generally used to store big data and real-time web applications.

Why NoSQL Database?

  • Large volumes of structured, semi-structured, and unstructured data
  • Agile sprints, quick iteration, and frequent code pushes
  • Object-oriented programming that is easy to use and flexible
  • Efficient, scale-out architecture

SQL vs NoSQL

SQL Database NoSQL Database
SQL databases are primarily called RDBMS or Relational Databases NoSQL databases are primarily called as Non-relational or distributed database
SQL databases are table-based databases NoSQL databases can be document-based, key-value pairs, or graph databases
MySQL uses SQL to query the database MongoDB uses BSON to query the database
Developed in the 1970s with a focus on reducing data duplication Developed in the late 2000s with a focus on scaling and rapid application change

Create Database

To create a database in MongoDB, use the following command:

use database_name;

Create Collection

Method 1: Insert to Create Collection

In MongoDB, you need not create a collection before inserting a document. The collection is created on the fly:

db.collection_name.insert(key: value, key: value…)

Method 2: Create Collection with Options

db.createCollection(name, options)

Example:

db.createCollection("PGprograms");

Documents

field1: value1, field2: value2, ...

Deleting the Collection

To drop a collection, first connect to the database in which you want to delete the collection and then type the following command to delete the collection:

db.collection_name.drop()

Note: Once you drop a collection, all the documents and the indexes associated with them will also be dropped. To preserve the indexes, use the remove() function that only removes the documents in the collection but doesn’t remove the collection itself and the indexes created on it.

Example:

db.gla.drop();

Create Documents

MongoDB stores data records as BSON documents. BSON is a binary representation of JSON documents. MongoDB documents are composed of field-and-value pairs and have the following structure:

field1: value1, field2: value2, ...
